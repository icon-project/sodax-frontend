# SODAX Partner Network

**Build cross-network applications. Skip the infrastructure.** SODAX coordinates complex DeFi execution across networks and liquidity as a single system, so partners can offer cross-network capabilities to their users without becoming a cross-network infrastructure team.

## System performance

- 10 intents per second (solver speed)
- 30,000 quote requests per second (system throughput)
- $10k swaps today, $50k swap capacity coming soon

## Built for builders — integration categories

- **Wallets** — let users swap and spend across networks without managing bridges or tracking asset locations. (Examples: Hana Wallet, MetaMask, Trust Wallet.)
- **DEXs and aggregators** — quote and execute cross-network swaps. Expand routing beyond single-network liquidity. (Examples: 1inch, Balanced, Uniswap.)
- **Lending protocols** — support multi-network user flows for collateral and borrowing across chains. (Examples: Aave, Compound, Venus.)
- **Perp DEXs and yield apps** — accept deposits from any supported network. Complete settlement into your native assets. (Examples: Amped Finance, GMX, dYdX.)
- **New networks** — launch with ready-made cross-network capabilities and liquidity access from day one. (Examples: LightLink, Sonic, new L2s.)
- **Solver marketplaces** — add efficient cross-network routes. Benefit end users across popular and exotic pairs. (Examples: Near Intents, 1inch Fusion.)

## What SODAX handles

- **Execution coordination** — SODAX coordinates liquidity, timing, and failure modes. Cross-network routing, failure recovery, async execution. Ship cross-network features in days instead of months.
- **EVM and non-EVM support** — connect to Ethereum, Solana, Sui, Stellar, and other environments through a single integration surface. 18+ networks. SODAX manages network-specific adapters.
- **Multi-bridge architecture** — integrates with established bridge protocols like LayerZero (OFT) and CCTP, plus purpose-built infrastructure with distributed custody. Designed to complement existing investments, not replace them.
- **Modular integration** — use only the components you need. Built on audited code and battle-tested primitives, integrated through composable SDKs.

## Three-layer SDK architecture

1. **Foundation layer** (`@sodax/sdk`) — core logic, swaps, lending, and bridging primitives.
2. **Connection layer** (`@sodax/wallet-sdk-react`) — opinionated React wrapper for cross-chain wallet providers.
3. **Experience layer** (`@sodax/dapp-kit`) — pre-built UI components and hooks for rapid deployment.

All layers provide access to SODAX-coordinated execution and protocol-supported liquidity.
